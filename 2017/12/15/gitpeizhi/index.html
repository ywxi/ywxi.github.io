git全局配置和单个仓库的用户名邮箱配置

用户名和邮箱的作用：用户名和邮箱地址是本地git客户端的一个变量，不会随git库而改变。每次commit都会用用户名和邮箱做记录。

一般在用git的时候，我们都配置了一个全局的用户名和邮箱：

如果公司的项目是放在自建的gitlab上面，如果你不进行配置用户名和邮箱的话，则会使用全局的，这个时候就会提交失败。正确的做法是针对公司的单个项目，在项目的根目录下进行单独配置。

git config

config配置有system级别、global(用户级别)和local(当前仓库)三个。设置先从system->global->local,后面的配置会覆盖前面的配置，分别使用--system/global/local可以查看相关的配置。

如果想忽略一些文件不提交，可以在根目录下建一个.gitignore文件。把不需要提交的文件添加到此处即可，这样就追踪不到这些文件了，就可以使用git add .把想提交的文件commit上去。
